0. Executive So What

Overall System Risk: Medium-High
  • Financial/operational outputs depend on backend, object storage, and asynchronous AI processing integrity.
  • Multiple guarded paths (upload validation, pending-folder filters, report generation) can block throughput if misconfigured.
  • Cross-stack coupling (Angular + Spring + MariaDB + OCI + Python YOLO) increases operational sensitivity.

Recommendation: prioritize reliability testing on upload, AI pending-folder consumption, and report generation/download routes before broad rollout.

1. One-Line Route Summary

Auth + Session: JWT-gated entry establishes user-context for client/project/report operations.
Media Upload: ZIP upload validates client/project ownership and seeds asynchronous storage + tracking logs.
Pending Folder Selection: Python pipeline selects non-processed project folders via DB flags and UUID links.
YOLO Inference: Multiple .pt models classify deterioration patterns from extracted images.
Report Synthesis: TXT/image content is merged with computed KPIs into versioned XLSX outputs.
Delivery: Completed reports stream back to frontend through project report download endpoint.

3. Route-by-Route Detailed Assessment

Route A - Media Upload and Validation
  • Entry condition: authenticated user uploads ZIP via /api/media/upload with clientId and projectId.
  • Primary intent: ingest survey artifacts and initialize upload tracking.
  • Main checks: file type/size validation + client/project existence and ownership.
  • Success: UploadLog created and async transfer initiated.
  • Failure: validation/entity errors stop ingest early.
  • Risk Level: Medium
  • Reason: strict ownership and artifact checks can block throughput when metadata is inconsistent.
  • Business Impact: ingestion failure prevents downstream AI analysis and report timelines.
Route B - AI Pending-Folder Pull and Object Download
  • Entry condition: Python main loop fetches folders where analysis is pending and artifact not yet downloaded.
  • Primary intent: synchronize cloud ZIP artifacts into local processing pipeline.
  • Main checks: uuid/object existence, download progress, mark_as_downloaded update.
  • Success: zipped tramo assets become locally available for extraction and inference.
  • Failure: object/storage connection issues stall AI batch progress.
  • Risk Level: High
  • Reason: asynchronous cloud dependency with DB-state coupling.
  • Business Impact: delayed condition assessment and backlog accumulation.
Route C - YOLO Inference and Metrics Generation
  • Entry condition: extracted image folders indexed by UUID.
  • Primary intent: detect deterioration classes and produce measured data.
  • Main checks: model loading, image parse consistency, per-folder processing completeness.
  • Success: inferred damage outputs become available to report aggregation.
  • Failure: malformed image sets or model/runtime mismatch reduce result quality/completeness.
  • Risk Level: High
  • Reason: model-driven route is central to value proposition and output credibility.
  • Business Impact: inaccurate or delayed findings affect maintenance prioritization decisions.
Route D - Report Generation and Distribution
  • Entry condition: project report generation call with available txt/image evidence.
  • Primary intent: produce versioned XLSX outputs for client consumption.
  • Main checks: txt presence, wk_reporte upsert, template fill, storage upload completion.
  • Success: reportes row completed with route/file metadata and streamable artifact.
  • Failure: missing txt or stream/storage errors interrupt delivery path.
  • Risk Level: Medium
  • Reason: deterministic process but sensitive to artifact integrity.
  • Business Impact: report delays defer executive road-condition review and planning.
